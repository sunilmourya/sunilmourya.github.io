I want to write something honest here, because I keep seeing the sanitized version of this conversation online and it doesn't match my actual experience.

## The thing nobody says

AI coding tools are useful. I'm not writing an anti-AI essay. I use them for documentation lookup, code review, understanding unfamiliar codebases, generating boilerplate.

But over the last 18 months, I noticed something. My tolerance for being stuck — genuinely stuck, not knowing what to do next — dropped significantly. Not because the problems got harder. Because I stopped practicing it.

When I'm stuck now, the reflex is: tab to the assistant, describe the problem, receive a solution. Time to resolution: 90 seconds. Dopamine: immediate. Muscle memory formed: zero.

When I was building C++ depth, there was no tab to press. There was just the problem and time. I once spent three days on a memory corruption bug in an audio DSP pipeline — a single off-by-one in a ring buffer index. Three days. I can still feel the exact shape of that bug. I will never write that class of error again.

The C++ knowledge is *in* me in a way the Rust knowledge isn't yet. Because the C++ knowledge was forged in friction.

## The atrophy that hides

Here's the part that's hard to see in real time: the atrophy doesn't feel like atrophy. It feels like efficiency.

"I used AI to solve this in 10 minutes instead of 3 hours." True. Also: I now have 10 minutes of experience instead of 3 hours of it.

The 3-hour version would have built something. Pattern recognition. Intuition about what goes wrong and why. The visceral sense of the language's model. The 10-minute version built a result.

Results aren't useless. But results aren't the same as capability.

## What I'm actually doing about it

The 100-day challenge isn't performance. It's an experiment with one hypothesis: that the muscle memory I want requires the friction I've been avoiding.

The rules are specific because vague rules evaporate. "I'll use AI less" is not a rule. "30-minute minimum struggle window, timer on the desk, AI closed" is a rule.

I'm also paying attention to what I'm *not* doing. I'm not swearing off AI tools — that would be theatrical and probably counterproductive. I'm restricting them during the specific phase where pattern formation happens.

There's a difference between a surgeon using a scalpel and a surgical student using one. The student needs to develop hand feel, mistake recognition, material intuition. The AI is the scalpel. The question is whether I'm the surgeon or still the student.

## One more thing

I'm writing this publicly partly to create accountability, but mostly because I suspect I'm not the only person who has felt this and not said it.

The narrative around AI and learning is mostly either "AI will replace developers" or "AI is just a tool, developers are fine." Neither of those captures the subtler thing that's actually happening to the craft of building intuition.

The subtler thing is worth naming.
