## Why I'm writing this

Tomorrow starts 100 days of deliberate practice. But I've done "100 days" things before and they dissolved into streaks I was just checking off, not building anything real.

So today — the day before — I'm writing the rules. The honest version. The one I'll come back to on Day 37 when it gets boring.

## The problem I'm solving

When I was building C++ depth years ago, I spent *days* stuck on problems. Stack traces at 2am. Memory I was forced to build because there was no other way.

With Rust, I have AI tools. And I've been using them too early — reaching for autocomplete before my brain has had a chance to form the pattern. The muscle memory never builds. Three years in and I still feel like I'm borrowing someone else's understanding.

That stops tomorrow.

## The rules

**For Rust (non-negotiable):**
- Minimum 30-minute struggle window before asking AI anything
- Core concepts — ownership, lifetimes, trait bounds, async — are AI-free zones, full stop
- Plain editor (Helix, no Copilot) for all exercises
- AI is for code *review*, not code *generation*
- If I used AI to write something, I rewrite it from memory the next day

**For C++:**
- One deep topic per week, not breadth
- Connect it to Rust wherever possible (FFI, unsafe patterns, memory models)
- Document the comparison — both languages on the same problem

**For Running:**
- 4 runs per week, non-negotiable
- Log every run, even the bad ones
- Long run on Sundays

**For the blog:**
- Write daily. Even if it's 3 lines.
- Honest logs — stuck counts as content, not failure

## What 100 days looks like

| Phase | Days | Focus |
|-------|------|-------|
| Foundations | 1–20 | Rust ownership from scratch, Rustlings |
| DSA | 21–45 | Linked list, BST, graph — no stdlib |
| Systems | 46–65 | CLI tools, real projects, FFI |
| Interview prep | 66–80 | System design, patterns |
| SDV niche | 81–100 | Automotive Rust, audio stack |

Running target: half-marathon by Day 90.

## What success looks like

Not "I completed 100 days." That's not it.

Success is: on Day 101, I can implement a doubly-linked list in Rust with correct lifetimes, from memory, in a plain editor, without help. And run 21km without walking.

That's the bar.
